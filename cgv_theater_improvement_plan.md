# CGV 극장 선택 개선 기획서 — MP2 Day 1 데모 예시

> 구조: 배경(미션) → 가설 → 목업 → 실험계획 → 기대결과
> 근거 출처: 2026-06-12 cgv.co.kr 모바일웹 직접 답사 (홈 → 결제 직전까지 전체 퍼널)

---

## 1. 배경 (미션)

**CGV의 공식 미션**: "생활 속 깊이 있는 문화 활동을 통해 건강하고 즐거운 삶에 기여" / "고객에게 잊을 수 없는 감동을 선사합니다." (corp.cgv.co.kr)

이 문장은 슬로건으로는 좋지만 제품 판단 기준으로는 너무 넓다. 제품팀이 쓸 수 있는 **내부 미션**으로 좁히면:

> "관객이 보고 싶은 영화를 발견한 순간부터 상영관 좌석에 앉기까지의 여정을 가장 매끄럽게 만든다."

CGV는 사용자가 **몸을 움직여 극장까지 가야** 하는 서비스다. 따라서 앱의 핵심 역할은 예매 퍼널(영화 → 극장 → 시간 → 좌석 → 결제)을 막힘없이 통과시키는 것.

**문제 발견 (직접 답사)**: 예매 퍼널 2단계 "극장 선택"이 행정·상권명 텍스트 리스트(서울 > 강남/강변/고덕강일/등촌…)로 되어 있다.

- 사용자의 실제 결정 기준: "지금 내 위치에서 가깝고 빨리 갈 수 있는가"
- UI가 요구하는 것: "서울 상권 지명을 외우고 있는가"
- → **결정 기준과 UI의 불일치**. '고덕강일'이 어딘지 모르는 사용자는 여기서 막힌다.
- 위치 기반 추천 없음, 지도 없음, 최근 방문 극장 고정 없음 (검색은 지명 텍스트 검색만 존재)

## 2. 가설

**지표 문제 가정 (Y)**
극장 선택 단계의 소요시간이 길고, 극장선택 → 시간표 전환율이 낮을 것이다.
특히 신규 사용자, 이사·여행 등 낯선 지역에 있는 사용자 세그먼트에서 두드러질 것이다.
*(실데이터 없음 — 휴리스틱 답사 + VOC로 세운 가정임을 명시)*

**개선 가설 (A)**
> 극장을 지명 리스트가 아니라 **내 위치 기준 거리·이동시간**으로 보여주면
> (가까운 순 자동 정렬 + 지도 보기 토글 + 최근 방문 극장 상단 고정),
> 극장 선택 소요시간이 줄고 단계 전환율이 오를 것이다.

**메커니즘**: ① 지명→위치 변환이라는 머릿속 작업을 UI가 대신 (인지 부하 제거) ② 거리·시간이 보이면 선택 확신이 생겨 진행 속도 증가.

**반대논리 (역할극 방어)**
- 헤비유저는 단골 극장이 있어 현행 리스트가 더 빠를 수 있다 → 그래서 "지도 우선"이 아니라 **"위치 우선"**: 기본은 가까운 순 리스트 + 최근 극장 고정, 지도는 보조 토글
- 위치 권한 팝업 자체가 새 이탈 지점 → 권한 거부 시 현행 리스트 폴백 (Edge Case)
- 지도 SDK 비용·로딩 속도 (Dependency)

## 3. 목업

작동하는 프로토타입: `cgv_theater_picker_prototype.html` (같은 폴더)

- **A/B 토글**: 현행 방식(대조군) ↔ 개선안(실험군)을 한 화면에서 비교
- 개선안: 내 위치(역삼역 가정) 기준 거리순 정렬 + 도보/지하철 시간 + 최근 방문 극장 고정 + 지도 뷰 토글
- 더미데이터: 서울 CGV 10개 지점 좌표·거리 (dummy-dataset 스킬 활용 예시)

## 4. 실험계획

| 항목 | 내용 |
|------|------|
| 설계 | A/B 테스트 — 대조군: 현행 지역 리스트 / 실험군: 위치 기반 정렬 + 지도 토글 + 최근 극장 고정 |
| 핵심 지표 | ① 극장 선택 단계 소요시간(중앙값) ② 극장선택→시간표 전환율 ③ **전체 예매 완료율** (이탈이 다른 단계로 이동했는지 확인) |
| 가드레일 | ① 위치 권한 팝업으로 인한 단계 진입 이탈률 ② 극장 변경률(잘못 선택해서 되돌아오는 비율) |
| 세그먼트 | 신규 vs 재방문 / 자주 가는 극장 보유 여부 / 위치 권한 허용 여부 |
| 판정 기준 | 전체 예매 완료율이 통계적으로 유의하게 상승 + 가드레일 비악화 시 전체 적용 |
| 수강생 버전 | 실제 A/B 불가 → ① 실험설계서 작성 ② 실험군을 더미데이터 웹앱으로 구현 ③ 동료 5명에게 "ㅇㅇ역에서 군체 가장 빨리 보기" 과제로 양쪽 소요시간 측정 (프록시 검증) |

## 5. 기대결과

- 극장 선택 소요시간 단축 (목표 예: 중앙값 30% 감소 — 사용성 테스트로 추정치 보정)
- 극장선택→시간표 전환율 상승, 전체 예매 완료율 상승
- 신규·낯선 지역 사용자 세그먼트에서 효과 최대 → 신규 사용자 첫 예매 성공률 개선
- 미션 관점: "발견 → 좌석" 여정의 마찰 제거 = 내부 미션에 직접 기여
- 가드레일 비악화 확인 시: 최근 극장 고정은 헤비유저에게도 순이익
